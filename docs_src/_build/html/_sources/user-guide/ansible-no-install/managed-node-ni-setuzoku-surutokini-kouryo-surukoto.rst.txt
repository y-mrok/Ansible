##################################################
管理対象ノードに接続するときに考慮すること
##################################################
コントロールノードから管理対象ノードに接続するときに考慮すべき事項です。

- 管理対象ノードのフィンガープリントを無視するか否か
- 認証方式はパスワード認証か鍵認証か

まず、フィンガープリントの扱いです。フィンガープリントを無視するのはセキュリティの観点から問題があります。しかし、すべての管理対象ノードがイントラネット内にある場合、フィンガープリントを無視するかどうかは検討の余地があります。フィンガープリントの受け入れ確認時

.. code-block:: none

   Are you sure you want to continue connecting (yes/no/[fingerprint])?
   
に常に :guilabel:`yes` と回答しているのであれば無視して問題ないのでは？　と思います。

次に認証方式です。セキュリティを優先するなら鍵認証、手軽さを優先するならパスワード認証です。状況に応じて、どちらを使用するか決定すべきです。
