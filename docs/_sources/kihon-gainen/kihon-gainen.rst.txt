##################################################
基本概念
##################################################

- コントロールノード　：　Control node
   - Ansible がインストールされた端末のこと
   - Python をインストールした Linux , FreeBSD , Solaris , macOS などの端末をコントロールノードに使用できるが、 Windows 端末はコントロールノードに使用できない
   - コントロールノード上で ``ansible`` コマンドや ``ansible-playbook`` コマンドを使用して Ansible のコマンドやプレイブックを実行する

- 管理対象ノード　：　Managed node
   - Ansible で管理される Linux , FreeBSD , Solaris , macOS , Windows などの端末（サーバー）やネットワーク機器のこと
   - 単に「ホスト」と呼ばれることもある

- インベントリ　：　Inventory
   - 管理対象ノードのリスト
   - インベントリファイルは「ホストファイル」と呼ぶこともある
   - 管理対象ノードに接続するための情報（ IP アドレス、ログインアカウント ．．．）などを指定する
   - 管理対象ノードをまとめてグループを作成できる

- コレクション　：　Collections
   - Ansible で作成したコンテンツの配布形式
   - プレイブック、ロール、モジュール、プラグインなどがある
   - ユーザーの管理や VLAN の管理など特定の用途を持つ
   - タスクで 1 つのモジュールを呼び出したり、プレイブックで複数のモジュールを呼び出したりできる

- タスク　：　Tasks
   - Ansible のアクションの単位
   - アドホックコマンドで 1 つのタスクを 1 回実行できる

- プレイブック　：　Playbooks
   - タスクのリスト
   - プレイブックファイルとして保存されるため、繰り返し使用（実行）できる
   - YAML で記述される
