ansible.builtin.import_tasks - タスクを静的に取り込む¶

概要¶

  • 別ファイルに定義したタスクをプレイに静的に取り込みます。

主なパラメーター¶

パラメーター

選択肢／デフォルト

説明

file

取り込むファイルのパスを指定する

使用例¶

別ファイルに定義したタスクをプレイ内に取り込む¶

インベントリです

---
all:
  hosts:
    hydrogen:

ファイル名を直接指定¶

別ファイルに定義したタスクをプレイ内に取り込みます。取り込むファイルは ansible.builtin.import_tasks に続けて指定します。

- name: Example of "ansible.builtin.import_tasks" usage
  hosts: all
  gather_facts: no

  tasks:
    - name: Message No.1
      ansible.builtin.debug:
    
    - name: Import tasks
      ansible.builtin.import_tasks: ./imports/imp_task1.yml

    - name: Message No.2
      ansible.builtin.debug:

取り込むタスクです。

# imports/imp_task1.yml
---
- name: List of tasks to import No.1
  ansible.builtin.debug:
    msg: "task #1"

- name: List of tasks to import No.2
  ansible.builtin.debug:
    msg: "task #2"

実行ログです。 ansible.builtin.import_tasks モジュールを含むタスクの位置で、取り込んだタスクを実行しました。 ansible.builtin.import_tasks モジュールを含むタスクの実行ログは出力されません。

“file” パラメーターを使用してファイル名を指定¶

file パラメーターを使用して、別ファイルに定義したタスクをプレイ内に取り込みます。取り込むタスクは「 ファイル名を直接指定 」と同じです。

- name: Example of "ansible.builtin.import_tasks" usage
  hosts: all
  gather_facts: no

  tasks:
    - name: Message No.1
      ansible.builtin.debug:
    
    - name: Import tasks
      ansible.builtin.import_tasks:
        file: ./imports/imp_task1.yml

    - name: Message No.2
      ansible.builtin.debug:

実行ログです。 ansible.builtin.import_tasks モジュールを含むタスクの位置で、取り込んだタスクを実行しました。 ansible.builtin.import_tasks モジュールを含むタスクの実行ログは出力されません。

ディレクティブ¶

become ディレクティブ¶

ansible.builtin.import_tasks モジュールを使用したタスクに become ディレクティブを適用すると、取り込むタスク内の各タスクに継承します。すなわち、取り込むタスクの各タスクに bocome ディレクティブを設定したのと同じ動作をします。

プレイです。

- name: Example of "ansible.builtin.import_tasks" usage
  hosts: all
  gather_facts: no

  tasks:
    - name: Message No.1
      ansible.builtin.debug:
    
    - name: import tasks
      ansible.builtin.import_tasks: ./imports/imp_task4.yml
      become: yes

    - name: Message No.2
      ansible.builtin.debug:

取り込むタスクです。

# imports/imp_task4.yml
---
- name: Register a user account.
  ansible.builtin.user:
    name: "hanako"
    state: present

- name: Install Apache
  ansible.builtin.dnf:
    name: httpd
    state: present

実行ログです。 ansible.builtin.user モジュールと ansible.builtin.dnf モジュールはともに管理者権限が必要なモジュールですが各タスクに become ディレクティブは設定していません。しかし、タスク [import tasks] に設定した become ディレクティブを継承したため、正常に実行しました。

piyo@vmpc:~/example/collection$ ansible-playbook -i hosts.yml sample11.yml

when ディレクティブ¶

ansible.builtin.import_tasks モジュールを使用したタスクに when ディレクティブを適用すると、取り込むタスク内の各タスクに継承します。すなわち、取り込むタスクの各タスクに when ディレクティブを設定したのと同じ動作をします。

インベントリです。

---
all:
  hosts:
    centos83:
    ubuntu2004:

プレイです。

- name: Example of "ansible.builtin.import_tasks" usage
  hosts: all

  tasks:
    - name: Message No.1
      ansible.builtin.debug:
    
    - name: import tasks
      ansible.builtin.import_tasks: ./imports/imp_task1.yml
      when: ansible_facts['os_family'] == "Debian"

    - name: Message No.2
      ansible.builtin.debug:

取り込むタスクです。

# imports/imp_task1.yml
---
- name: List of tasks to import No.1
  ansible.builtin.debug:
    msg: "task #1"

- name: List of tasks to import No.2
  ansible.builtin.debug:
    msg: "task #2"

実行ログです。 ansible.builtin.import_tasks モジュールを使用したタスクに設定した when ディレクティブを取り込むタスク内の各タスクが継承しました。その結果、取り込むタスクの各タスクの実行結果に skipping などを表示しました。

loop ディレクティブ¶

loop ディレクティブは ansible.builtin.import_tasks モジュールを使用したタスク自体に適用できません。

プレイです。

- name: Example of "ansible.builtin.import_tasks" usage
  hosts: all
  gather_facts: no

  tasks:
    - name: Message No.1
      ansible.builtin.debug:
    
    - name: import tasks
      ansible.builtin.import_tasks: ./imports/imp_task2.yml
      loop:
        - taro
        - jiro
        - hanako

    - name: Message No.2
      ansible.builtin.debug:

実行ログです。 loop ディレクティブは使用できない旨のメッセージが表示されました。

piyo@vmpc:~/example/collection$ ansible-playbook -i hosts.yml sample13.yml 
ERROR! You cannot use loops on 'import_tasks' statements. You should use 'include_tasks' instead.

The error appears to be in '/home/piyo/example/collection/sample13.yml': line 9, column 7, but may
be elsewhere in the file depending on the exact syntax problem.

The offending line appears to be:


    - name: import tasks
      ^ here
piyo@vmpc:~/example/collection$

tags ディレクティブ¶

ansible.builtin.import_tasks モジュールを使用したタスクに tags ディレクティブを適用すると、取り込むタスク内の各タスクに継承します。すなわち、取り込むタスクの各タスクに tags ディレクティブを設定したのと同じ動作をします。

プレイです。

- name: Example of "ansible.builtin.import_tasks" usage
  hosts: all
  gather_facts: no

  tasks:
    - name: Message No.1
      ansible.builtin.debug:
    
    - name: import tasks
      ansible.builtin.import_tasks: ./imports/imp_task5.yml
      tags: job1

    - name: Message No.2
      ansible.builtin.debug:

取り込むタスクです。

# imports/imp_task5.yml
---
- name: role tasks No.1
  ansible.builtin.debug:
- name: role tasks No.2
  ansible.builtin.debug:
  tags: job2
- name: role tasks No.3
  ansible.builtin.debug:
  tags: job3

タグ job1 を指定した実行ログです。 ansible.builtin.import_tasks モジュールを使用したタスクに設定した tags ディレクティブを取り込むタスク内の各タスクが継承しました。

取り込むタスクのファイル名に変数を使用する¶

ファクト変数¶

ansible.builtin.import_tasks モジュールを使用したタスクで、取り込むタスクのファイル名にファクト変数は使用できません。

インベントリです。

---
all:
  hosts:
    centos83:
    ubuntu2004:

プレイです。

- name: Example of "ansible.builtin.import_tasks" usage
  hosts: all

  tasks:
    - name: Message No.1
      ansible.builtin.debug:
    
    - name: import tasks
      ansible.builtin.import_tasks: ./imports/imp_{{ ansible_facts['distribution'] }}.yml

    - name: Message No.2
      ansible.builtin.debug:

実行ログです。ファクト変数などはファイル名に使用できないと表示されました。

piyo@vmpc:~/example/collection$ ansible-playbook -i hosts2.yml sample14.yml 
ERROR! Error when evaluating variable in import path: ./imports/imp_{{ ansible_facts['distribution'] }}.yml.

When using static imports, ensure that any variables used in their names are defined in vars/vars_files
or extra-vars passed in from the command line. Static imports cannot use variables from facts or inventory
sources like group or host vars.
piyo@vmpc:~/example/collection$

vars セクション¶

ansible.builtin.import_tasks モジュールを使用したタスクで、取り込むタスクのファイル名を vars セクションで定義できます。

インベントリです。

---
all:
  hosts:
    hydrogen:

プレイです。

- name: Example of "ansible.builtin.import_tasks" usage
  hosts: all
  gather_facts: no

  vars:
    import_filename: ./imports/imp_task1.yml

  tasks:
    - name: Message No.1
      ansible.builtin.debug:
    
    - name: import tasks
      ansible.builtin.import_tasks: "{{ import_filename }}"

    - name: Message No.2
      ansible.builtin.debug:

取り込むタスクです。

# imports/imp_task1.yml
---
- name: List of tasks to import No.1
  ansible.builtin.debug:
    msg: "task #1"

- name: List of tasks to import No.2
  ansible.builtin.debug:
    msg: "task #2"

実行ログです。 vars セクションで定義したファイル名で指定したタスクを取り込んで実行しました。
