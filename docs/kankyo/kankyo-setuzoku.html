コントロールノードから管理対象ノードに接続する準備¶

コントロールノードから管理対象ノードに公開鍵認証方式で接続するための設定を行います。

手順¶

  1. 鍵を作成

  2. 管理対象ノードにアカウント ansible-user を作成

  3. 管理対象ノードに公開鍵を登録

  4. 管理対象ノードでアカウント ansible-user が管理者権限でコマンドを実行できるよう設定（ sudo の設定）

プレイブック¶

「 手順 」の内容のプレイを作成・実行します。

インベントリファイル¶

# hosts.yml
---
all:
  vars:
    ansible_user: vagrant
    ansible_password: vagrant
  hosts:
    hydrogen:
      ansible_host: 192.168.23.11
    helium:
      ansible_host: 192.168.23.12
    lithium:
      ansible_host: 192.168.23.13

プレイブックファイル¶

# node.yml
---
- name: 管理対象ノードに接続用のアカウントを作成
  hosts: all
  gather_facts: no

  vars:
    account: ansible-user

  tasks:   
    - name: 鍵を作成
      community.crypto.openssh_keypair:
        path: ~/.ssh/id_ed25519
        type: ed25519
        state: present
      run_once: yes
      delegate_to: localhost
    
    - name: アカウント ansible-user を作成
      ansible.builtin.user:
        name: "{{ account }}"
      become: yes
    
    - name: 管理対象ノードに鍵を登録
      ansible.posix.authorized_key:
        user: "{{ account }}"
        state: present
        key: "{{ lookup('file', '~/.ssh/id_ed25519.pub') }}"
      become: yes
    
    - name: sudo 用ファイルをコピー
      ansible.builtin.copy:
        src: ./sudo.txt
        dest: /etc/sudoers.d/ansible-user
        owner: root
        group: root
        mode: '0440'
      become: yes

sudo 設定用ファイル¶

ansible-user ALL=(ALL:ALL) NOPASSWD: ALL

コマンド¶

ansible-playbook -i hosts.yml node.yml
