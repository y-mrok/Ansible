4 つのセクション¶

targets セクション¶

  • 必須のセクション

  • プレイの実行対象になる管理対象ノードを指定する

使用頻度が高いディレクティブ¶

ディレクティブ

説明
hosts
※必須※

プレイの実行対象になる管理対象ノードまたはグループを指定する

become
プレイ内のすべてのタスクを管理者権限で実行するか否かを指定する
・yes：管理者権限で実行する
・no ：管理者権限で実行しない（デフォルト）

gather_facts
管理対象ノードのファクト変数を収集するか否かを指定する
・yes：収集する（デフォルト）
・no ：収集しない

記述例：

  • プレイを管理者権限で実行する

  • ファクト変数を収集しない

---
- name: Sample usage of hosts
  hosts: all
  become: yes
  gather_facts: no

hosts ディレクティブの管理対象ノードの指定パターン¶

  • すべての管理対象ノードを指定　→　all

  • 特定の管理対象ノードだけ指定　→　管理対象ノード名

  • グループを指定　→　グループに含まれるすべての管理対象ノードを指定したことと同じ

  • 管理対象ノードやグループを複数指定　→　, で連結

  • 複数のグループに共通して含まれる管理対象ノードを指定　→　グループを :& で連結

  • 複数のグループのすべての管理対象ノードを指定　→　グループを : で連結

使用するインベントリ

---
all:
  children:
    webservers:
      hosts:
        helium:
        lithium:
    dbservers:
      hosts:
        lithium:
    dnsservers:
      hosts:
        hydrogen:

  • すべての管理対象ノード

hosts: all

  • 管理対象ノードを 1 つだけ指定

hosts: hydrogen

  • 複数の管理対象ノードを指定

hosts: hydrogen, lithium

  • グループを指定

hosts: webservers

  • 複数のグループを指定

hosts: dbservers, dnsservers

  • 管理対象ノードとグループを指定

hosts: hydrogen, dbservers

  • 両方のグループに含まれる管理対象ノードだけを指定

hosts: webservers:&dbservers

  • 両方のグループに含まれるすべての管理対象ノードを指定

hosts: dbservers:dnsservers

vars セクション¶

  • プレイ内で使用する変数を指定（定義）する

  • 詳細は「 変数 」を参照

tasks セクション¶

  • 必須のセクション

  • 管理対象ノードで実行するタスクを シーケンス で記述する

  • タスクは記述した順に実行する

タスク¶

  • Ansible の実行単位で最小のもの

  • モジュールにパラメーターで値を与えて、何らかの実行をできるようにしたもの

Note

Ansible が用意しているすべてのモジュールは次のリンクで確認ください。

Collection Index

使用頻度が高いディレクティブ¶

ディレクティブ

説明

name

タスクに名前を付ける（ほぼ必須）

become
タスクを管理者権限で実行するか否かを指定する
・yes：管理者権限で実行する
・no ：管理者権限で実行しない（デフォルト）

when
条件を設定し、その成否でタスクを実行するか否かを判別する
・条件が成立：タスクを実行する
・条件が不成立：タスクの実行をスキップする

loop

タスクを繰り返し実行する

タスクの例¶

下記は アドホックコマンドで実行したもの をタスクに書き直した結果です。

tasks:
  - name: Install Apache
    ansible.builtin.dnf:
      name: httpd
      state: present
    become: yes
  - name: Enabling Apache services and starting services at startup
    ansible.builtin.systemd:
      name: httpd.service
      state: started
      enabled: yes
    become: yes
  - name: Open up the http service in the firewall
    ansible.posix.firewalld:
      service: http
      permanent: yes
      state: enabled
      immediate: yes
    become: yes

name ディレクティブから始まる 3 つのタスクが含まれています。

- name: Install Apache
  ansible.builtin.dnf:
    name: httpd
    state: present
  become: yes

- name: Enabling Apache services and starting services at startup
  ansible.builtin.systemd:
    name: httpd.service
    state: started
    enabled: yes
  become: yes

- name: Open up the http service in the firewall
  ansible.posix.firewalld:
    service: http
    permanent: yes
    state: enabled
    immediate: yes
  become: yes
