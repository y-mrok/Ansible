“changed” と判断する条件を定義する¶

  • changed_when ディレクティブでタスクを実行した結果、管理対象ノードの状態が変化された（ changed ）と判断する条件を定義する

  • changed_when ディレクティブで定義した条件式はタスクの実行後に評価する

  • 条件式の書式は when ディレクティブ と同じ

changed_when ディレクティブの使い方¶

changed_when ディレクティブを使用してタスクを実行した結果、管理対象ノードの状態が変更された（ changed ）と判断する条件を設定します。通常、タスクでモジュールを実行した場合、管理対象ノードが変更されたか否かは Ansible 側で適切に判断します。そのため、 changed_when ディレクティブは不要です。しかし、管理対象ノードでコマンドを実行する ansible.builtin.command モジュールとコマンドやシェルスクリプトを実行する ansible.builtin.shell モジュールの結果は常に changed です。この様なときは changed_when ディレティブで正しい実行結果（ ok や changed ）を返すよう設定することが望ましいです。 changed は ハンドラー を実行するトリガーになるため、正しい状態を返すことは大事です。

指定したユーザーアカウントが未作成のときに作成するシェルスクリプトです。

#!/bin/sh

grep $1 /etc/passwd
if [ $? -eq 0 ]
then
  exit 0 
elif [ $? -eq 1 ]
then
  sudo useradd $1
  exit 1
else
  exit 9
fi

管理対象ノードの状態とリターンコードの関係です。

リターンコード

管理対象ノードの状態

0

指定したユーザーアカウントがすでに登録されていた

1

指定したユーザーアカウントが未登録だったので登録した

9

上記の状態以外の状態が発生した

インベントリです。

---
all:
  hosts:
    hydrogen:

changed_when ディレクティブを使用していないプレイです。 “taro” と “jiro” は未登録のユーザーアカウントです。

---
- name: Example of error handling usage
  hosts: all
  gather_facts: no
 
  tasks:
    - name: Repeat for as many accounts as you need.
      ansible.builtin.include_tasks: ./include/user-add.yml
      loop: "{{ user_account }}"
      vars:
        user_account:
          - ansible-user
          - taro
          - jiro

# include/user-add.yml
---
- name: Create the specified user account when it has not yet been created.
  ansible.builtin.shell: "./user-add.sh {{ item }}"

実行ログです。アカウント “ansible-user” は登録済みユーザーなので管理対象ノードの状態は変わりません。しかし、実行結果は changed になっています。アカウント “taro” は未登録なのでリターンコードは 1 （ “rc”: 1 ）です。リターンコードがゼロ以外の場合はタスクの実行が失敗したと判断するのでエラーメッセージが表示されました。

piyo@vmpc:~/example/err$ ansible-playbook -i hosts.yml error16.yml 

PLAY [Example of error handling usage] ***************************************************************************************************************

TASK [Repeat for as many accounts as you need.] ******************************************************************************************************
included: /home/piyo/example/err/include/user-add.yml for hydrogen => (item=ansible-user)
included: /home/piyo/example/err/include/user-add.yml for hydrogen => (item=taro)
included: /home/piyo/example/err/include/user-add.yml for hydrogen => (item=jiro)

TASK [Create the specified user account when it has not yet been created.] ***************************************************************************
changed: [hydrogen]

TASK [Create the specified user account when it has not yet been created.] ***************************************************************************
fatal: [hydrogen]: FAILED! => {"changed": true, "cmd": "./user-add.sh taro", "delta": "0:00:00.470215", "end": "2021-04-11 12:48:33.665903", "msg": "non-zero return code", "rc": 1, "start": "2021-04-11 12:48:33.195688", "stderr": "", "stderr_lines": [], "stdout": "", "stdout_lines": []}

PLAY RECAP *******************************************************************************************************************************************
hydrogen                   : ok=4    changed=1    unreachable=0    failed=1    skipped=0    rescued=0    ignored=0   

piyo@vmpc:~/example/err$

./include/user-add.yml ファイルの内容を書き直し、ユーザーアカウントを登録したときだけ changed になるようにします。

# include/user-add.yml
---
- name: Create the specified user account when it has not yet been created.
  ansible.builtin.shell: "./user-add.sh {{ item }}"  
  register: user_status
  changed_when: user_status['rc'] == 1
  failed_when: user_status['rc'] > 1

changed_when ディレクティブとレジスター変数を使用し、タスクを実行した後のリターンコードが 1 のときだけ changed を表示します。 failed_when ディレクティブを併用し、リターンコードがゼロ以外のときにエラーと判断されることを避けています。

実行ログです。登録ユーザーの “ansible-user” は ok 、未登録ユーザーの “taro” と “jiro” は changed になっています。

piyo@vmpc:~/example/err$ ansible-playbook -i hosts.yml error16.yml 

PLAY [Example of error handling usage] ***************************************************************************************************************

TASK [Repeat for as many accounts as you need.] ******************************************************************************************************
included: /home/piyo/example/err/include/user-add.yml for hydrogen => (item=ansible-user)
included: /home/piyo/example/err/include/user-add.yml for hydrogen => (item=taro)
included: /home/piyo/example/err/include/user-add.yml for hydrogen => (item=jiro)

TASK [Create the specified user account when it has not yet been created.] ***************************************************************************
ok: [hydrogen]

TASK [Create the specified user account when it has not yet been created.] ***************************************************************************
changed: [hydrogen]

TASK [Create the specified user account when it has not yet been created.] ***************************************************************************
changed: [hydrogen]

PLAY RECAP *******************************************************************************************************************************************
hydrogen                   : ok=6    changed=2    unreachable=0    failed=0    skipped=0    rescued=0    ignored=0   

piyo@vmpc:~/example/err$

実行結果を常に “ok” にする¶

ansible.builtin.command モジュールや ansible.builtin.shell モジュールで実行するコマンドやシェルスクリプトが管理対象ノードの状態を変更しないことが明らかな場合、 changed_when ディレクティブを使用してタスクの実行結果を常に ok にできます。

/etc/passwd ファイルの内容を表示するタスクです。

---
- name: Example of error handling usage
  hosts: all
  gather_facts: no
 
  tasks:
    - name: Display the contents of the "/etc/passwd" file
      ansible.builtin.command: cat /etc/passwd

実行結果です。 ansible.builtin.command モジュールの仕様により、実行結果は changed です。

piyo@vmpc:~/example/err$ ansible-playbook -i hosts.yml error17.yml 

PLAY [Example of error handling usage] ***************************************************************************************************************

TASK [Display the contents of the "/etc/passwd" file] ************************************************************************************************
changed: [hydrogen]

PLAY RECAP *******************************************************************************************************************************************
hydrogen                   : ok=1    changed=1    unreachable=0    failed=0    skipped=0    rescued=0    ignored=0   

piyo@vmpc:~/example/err$

changed_when ディレクティブを使用して、タスクの実行結果が常に ok になるよう書き直したプレイです。 changed_when ディレクティブに “no” を設定すると、タスクの実行結果は常に ok になります。

---
- name: Example of error handling usage
  hosts: all
  gather_facts: no
 
  tasks:
    - name: Display the contents of the "/etc/passwd" file
      ansible.builtin.command: cat /etc/passwd
      changed_when: no

実行ログです。タスクに ansible.builtin.command モジュールを使用しましたが、実行結果は ok です。

piyo@vmpc:~/example/err$ ansible-playbook -i hosts.yml error18.yml 

PLAY [Example of error handling usage] ***************************************************************************************************************

TASK [Display the contents of the "/etc/passwd" file] ************************************************************************************************
ok: [hydrogen]

PLAY RECAP *******************************************************************************************************************************************
hydrogen                   : ok=1    changed=0    unreachable=0    failed=0    skipped=0    rescued=0    ignored=0   

piyo@vmpc:~/example/err$
