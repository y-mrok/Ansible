変数の参照方法¶

マッピングの参照¶

  • プレイ内で変数を参照するときは {{ と }} でくくる

  • パラメーターの設定値に変数を使用するときなど : の後で変数を参照するときは "{{ と }}" または '{{ と }}' でくくる

---
- name: Example of variable reference
  hosts: all
  gather_facts: no

  vars:
    source_path: ~/textfile.txt         # {{ source_path }}
    destination_path: /tmp/sample/      # {{ destination_path }}

  tasks:
    - name: Copying a file from a control node to a managed node
      ansible.builtin.copy:
        src: "{{ source_path }}"
        dest: "{{ destination_path }}"
    - name: View source and destination
      ansible.builtin.debug:
        msg: "コピー元のファイルは  {{ source_path }} です。コピー先は {{ destination_path }} です。"

マッピングをネストした場合の参照¶

  • 「 マッピングの参照 」のルール　＋　下位の変数を [' と '] でくくって参照する

---
- name: Example of variable reference
  hosts: all
  gather_facts: no

  vars:
    file_path:
      source_path: ~/textfile.txt       # file_path['source_path']
      destination_path: /tmp/sample/    # file_path['destination_path']

  tasks:
    - name: Copying a file from a control node to a managed node
      ansible.builtin.copy:
        src: "{{ file_path['source_path'] }}"
        dest: "{{ file_path['destination_path'] }}"
    - name: View source and destination
      ansible.builtin.debug:
        msg: "コピー元のファイルは  {{ file_path['source_path'] }} です。コピー先は {{ file_path['destination_path'] }} です。"

マッピングの中にシーケンスをネストした場合の参照¶

  • 「 マッピングの参照 」のルール　＋　 要素番号（ゼロオリジン）を [ と ] でくくって参照する

---
- name: Example of variable reference
  hosts: all
  gather_facts: no

  vars:
    file_path:
      - ~/textfile.txt                  # file_path[0]
      - /tmp/sample/                    # file_path[1]

  tasks:
    - name: Copying a file from a control node to a managed node
      ansible.builtin.copy:
        src: "{{ file_path[0] }}"
        dest: "{{ file_path[1] }}"
    - name: View source and destination
      ansible.builtin.debug:
        msg: "コピー元のファイルは  {{ file_path[0] }} です。コピー先は {{ file_path[1] }} です。"

マッピングの中にシーケンスをネストし、シーケンスの中にマッピングをネストした場合の参照¶

---
- name: Example of variable reference
  hosts: all
  gather_facts: no

  vars:
     user_account:
       - user_name: hanako-san                    # user_account[0]['user_name']
         user_password: hanako-no-password        # user_account[0]['user_password']
       - user_name: taro-san                      # user_account[1['user_name']
         user_password: taro-no-password          # user_account[1]['user_password']

  tasks:
    - name: Create the first account
      ansible.builtin.user:
        name: "{{ user_account[0]['user_name'] }}"
        password: "{{ user_account[0]['user_password'] | password_hash('sha512') }}"
        state: present
      become: yes
    - name: Create the second account
      ansible.builtin.user:
        name: "{{ user_account[1]['user_name'] }}"
        password: "{{ user_account[1]['user_password'] | password_hash('sha512') }}"
        state: present
      become: yes
    - name: Check the first person's account
      ansible.builtin.debug:
        msg: "アカウント {{ user_account[0]['user_name'] }} のパスワードは {{ user_account[0]['user_password'] }} です。"
    - name: Check the second person's account
      ansible.builtin.debug:
        msg: "アカウント {{ user_account[1]['user_name'] }} のパスワードは {{ user_account[1]['user_password'] }} です。"

ファクト変数の参照¶

  • 基本は上述の 4 パターンと同じ

  • 最上位の変数名は ansible_facts

  • 下位の変数を参照するとき、プレフィックスの ansible_ は削除する

ファクト変数の最初の部分です。

hydrogen | SUCCESS => {
    "ansible_facts": {
        "ansible_all_ipv4_addresses": [
            "192.168.23.11"
        ],
        "ansible_all_ipv6_addresses": [
            "fe80::8ac4:75a6:8719:7a"
        ],
        "ansible_apparmor": {
            "status": "disabled"
        },
        "ansible_architecture": "x86_64",

このように先頭は ansible_facts です。したがって、ファクト変数は次のように表記します。

ansible_facts['下位の変数名`]

  • ansible_all_ipv4_addresses

    "ansible_facts['all_ipv4_addresses'][0]"

  • ansible_apparmor 内の status

    ansible_facts['apparmor']['status']

  • ansible_architecture

    ansible_facts['architecture']
