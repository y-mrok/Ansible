lookup プラグイン¶

  • プレイブックから外部ソース（ファイル、データベース 等）にアクセスする機能を提供する

構文¶

lookup('プラグイン名', 引数)

使用例¶

file¶

プラグイン名

引数

動作

file

ファイルのパス

引数に指定したファイルの内容を取得する

テキストファイルの内容を読み取るプレイです。

---
- name: Examples of plugin usage.
  hosts: all
  gather_facts: no

  tasks:
    - name: Get the contents of a text file.
      ansible.builtin.debug:
        msg: "{{ contents }}"
      vars:
        contents: "{{ lookup('file', './testfile.txt') }}"

コントロールノード上の読み取るファイルの内容です。

abcdefg
hijklmn
1234557

実行ログです。改行コードは \n で表示されました。

piyo@vmpc:~/example/plugin$ ansible-playbook -i hosts.yml plugin1.yml 

PLAY [Examples of plugin usage.] *********************************************************************************************************************

TASK [Get the contents of a text file.] **************************************************************************************************************
ok: [hydrogen] => {
    "msg": "abcdefg\nhijklmn\n1234557"
}

PLAY RECAP *******************************************************************************************************************************************
hydrogen                   : ok=1    changed=0    unreachable=0    failed=0    skipped=0    rescued=0    ignored=0   

piyo@vmpc:~/example/plugin$

csvfile¶

プラグイン名

引数

動作

csvfile
検索する文字列
file= CSV ファイルのパス
delimiter=区切り文字（デフォルト：タブ文字）
col=取り出す値のカラム番号（ゼロオリジン, デフォルト：1 ）

CSV ファイルの最初のカラム（ col=0 ）の中から「検索する文字列」に合致する文字列を検索し、指定したカラム番号の項目の値を返す

アカウントを作成するプレイです。アカウントのパスワードとコメントは CSV ファイルから読み取ります。

---
- name: Examples of plugin usage.
  hosts: all
  gather_facts: no

  tasks:
    - name: Obtain the password from the CSV file.
      ansible.builtin.user:
        name: item
        password: "{{ lookup('csvfile', '{{ item }} file=./passwd.csv delimiter=,') | password_hash('sha512') }}"
        comment: "{{ lookup('csvfile', '{{ item }} file=./passwd.csv delimiter=, col=2') }}"
      loop: "{{ accounts }}"
      vars:
        accounts:
          - taro
          - jiro
          - hanako
      become: yes

コントロールノード上の読み取る CSV ファイルの内容です。

taro,taropass,Osaka Taro
jiro,jiropass,Kyoto Jiro
hanako,hanakopass,Kobe hanako

実行ログです。

piyo@vmpc:~/example/plugin$ ansible-playbook -i hosts.yml plugin2.yml

PLAY [Examples of plugin usage.] *********************************************************************************************************************

TASK [Obtain the password from the CSV file.] ********************************************************************************************************
changed: [hydrogen] => (item=taro)
changed: [hydrogen] => (item=jiro)
changed: [hydrogen] => (item=hanako)

PLAY RECAP *******************************************************************************************************************************************
hydrogen                   : ok=1    changed=1    unreachable=0    failed=0    skipped=0    rescued=0    ignored=0   

piyo@vmpc:~/example/plugin$

env¶

プラグイン名

引数

動作

env

取得する環境変数名

引数に指定した環境変数の値を取得する

環境変数 PATH の値を取得するプレイです。

---
- name: Examples of plugin usage.
  hosts: all
  gather_facts: no

  tasks:
    - name: Get the value of the PATH environment variable
      ansible.builtin.debug:
        msg: "{{ lookup('env', 'PATH') }}"

コントロールノードの環境変数 PATH の値です。

piyo@vmpc:~/example/plugin$ printenv | grep PATH

ini¶

プラグイン名

引数

動作

ini
検索するキー（文字列）
section=キーを検索するセクション
file= CSV ファイルのパス

ini ファイルから指定したキーの値を取得する

ini ファイルから製品版のユーザー名を取得するプレイです。

---
- name: Examples of plugin usage.
  hosts: all
  gather_facts: no

  tasks:
    - name: Get a user name for the product version
      ansible.builtin.debug:
        msg: "{{ lookup('ini', 'user section=production file=testfile.ini') }}"

コントロールノード上の読み取る ini ファイルの内容です。

[production]
user=produser
path=/usr/prod/path/file

[staging]
user=staguser
path=/usr/stag/path/file

実行ログです。指定したセクション（ production ）内の値を表示しました。

piyo@vmpc:~/example/plugin$ ansible-playbook -i hosts.yml plugin4.yml

PLAY [Examples of plugin usage.] *********************************************************************************************************************

TASK [Get a user name for the product version] *******************************************************************************************************
ok: [hydrogen] => {
    "msg": "produser"
}

PLAY RECAP *******************************************************************************************************************************************
hydrogen                   : ok=1    changed=0    unreachable=0    failed=0    skipped=0    rescued=0    ignored=0   

piyo@vmpc:~/example/plugin$
