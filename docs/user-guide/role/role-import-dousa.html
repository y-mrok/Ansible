ansible.builtin.import_role の動作¶

ディレクティブを併用して ansible.builtin.import_role モジュールの動作を確認します。

ロールの実行とディレクティブ¶

ディレクトリ構造です。

.
├── hosts2.yml
├── roles
│   └── basic_webserver_configuration2
│       ├── defaults
│       │   └── main.yml
│       ├── files
│       │   └── index.html
│       ├── handlers
│       │   └── main.yml
│       └── tasks
│           └── main.yml
└── webservers3.yml

インベントリの内容です。

---
all:
  hosts:
    centos83:
    ubuntu2004:

ロールの内容です。

  • defaults/main.yml

    ---
http_port: 8080
http_user: apache
http_group: apache
http_mode: "0775"
http_packages:
  - httpd
  - httpd-tools
  - httpd-devel
  - httpd-manual

  • files/

    .
├── hosts2.yml
├── roles
│   └── basic_webserver_configuration2
│       ├── defaults
│       │   └── main.yml
│       ├── files
│       │   └── index.html
│       ├── handlers
│       │   └── main.yml
│       └── tasks
│           └── main.yml
└── webservers3.yml

  • tasks/main.yml

    ---
- name: Install the latest version of the httpd package.
  ansible.builtin.dnf:
    name: "{{ http_packages }}"
    state: latest
  notify:
    - Installed the httpd package

- name: Set the server name for Apache.
  ansible.builtin.lineinfile:
    path: "/etc/httpd/conf/httpd.conf"
    regexp: "^#ServerName"
    line: "ServerName {{ ansible_facts['fqdn'] }}:{{ http_port }}"
    state: present
  notify:
    -  Edit the configuration file

- name: Set the http port for httpd.conf.
  ansible.builtin.lineinfile:
    path: "/etc/httpd/conf/httpd.conf"
    regexp: "^Listen "
    line: "Listen {{ http_port }}"
    state: present
  notify:
    -  Edit the configuration file

- name: Change the group and user for the /var/www/html directory.
  ansible.builtin.file:
    path: "/var/www/html"
    owner: "{{ http_user }}"
    group: "{{ http_group }}"
    mode: "{{ http_mode }}"
    state: directory

- name: Change the group and user for the /var/www/cgi-bin directory.
  ansible.builtin.file:
    path: "/var/www/cgi-bin"
    owner: "{{ http_user }}"
    group: "{{ http_group }}"
    mode: "{{ http_mode }}"
    state: directory

- name: Copying content.
  ansible.builtin.copy:
    src: "index.html"
    dest: "/var/www/html/"
    owner: "{{ http_user }}"
    group: "{{ http_group }}"
    mode: "{{ http_mode }}"

- name: Open a hole in the firewall of the CentOS management node for the http port.
  ansible.posix.firewalld:
    port: "{{ http_port }}/tcp"
    zone: public
    permanent: yes
    state: enabled
  notify:
    - Reload firewalld

  • handlers/main.yml

    ---
- name: Syntax check of httpd.conf.
  ansible.builtin.command: apachectl configtest 
  listen: 
    - Edit the configuration file
  become: yes

- name: Restart or start httpd.service.
  ansible.builtin.systemd:
    name: httpd.service
    enabled: yes
    state: restarted
  listen: 
    - Installed the httpd package
    - Edit the configuration file
  become: yes

- name: Reload the firewalld configuration.
  ansible.builtin.systemd:
    name: firewalld.service
    enabled: yes
    state: reloaded
  become: yes
  listen: 
    - Reload firewalld

ansible.builtin.import_role モジュールを使用したプレイと実行ログです。

---
- name: Example of using roles
  hosts: all

  tasks:
    - name: Web server construction roles.
      ansible.builtin.import_role:
        name: basic_webserver_configuration2
      when: ansible_facts['distribution'] == "CentOS"
      become: yes

ロール内のタスクの一部とその実行ログです。

- name: Install the latest version of the httpd package.
  ansible.builtin.dnf:
    name: "{{ http_packages }}"
    state: latest
  notify:
    - Installed the httpd package

TASK [basic_webserver_configuration2 : Install the latest version of the httpd package.] *************************************************************
skipping: [ubuntu2004]
changed: [centos83]

ansible.builtin.import_role モジュールに設定したディレクティブはロール内の各タスクに継承されます。すなわち、 ansible.builtin.import_role モジュールに設定した when ディレクティブと become ディレクティブはロール内の各タスクに継承されます。結果、管理対象ノード ubuntu2004 で ansible.builtin.dnf モジュールの実行は skipping になり、管理対象ノード centos83 でモジュールは正常に実行されました。

loop ディレクティブ¶

ディレクトリ構造です。

.
├── hosts.yml
├── roles
│   └── loop-test
│       └── tasks
│           └── main.yml
└── roles1.yml

ロールの内容です。

  • tasks/main.yml

    ---
- name: Display the start message
  ansible.builtin.debug:
    msg: "Start!"

- name: Display the contents of the variable item
  ansible.builtin.debug:
    msg: "The value of item is {{ item }}."

- name: Display an exit message
  ansible.builtin.debug:
    msg: "End!"

プレイの内容です。

---
- name: Example of using roles
  hosts: all

  tasks:
    - name: The ansible.builtin.import_role module with the loop directive set
      ansible.builtin.import_role:
        name: loop-test
      loop:
        - 1
        - 2
        - 3

実行ログです。 ansible.builtin.import_role モジュールに loop ディレクティブは設定できないのでエラーになりました。

piyo@vmpc:~/example/role$ ansible-playbook -i hosts.yml roles1.yml 
ERROR! You cannot use loops on 'import_role' statements. You should use 'include_role' instead.

The error appears to be in '/home/piyo/example/role/roles1.yml': line 6, column 7, but may
be elsewhere in the file depending on the exact syntax problem.

The offending line appears to be:

  tasks:
    - name: The ansible.builtin.import_role module with the loop directive set
      ^ here
piyo@vmpc:~/example/role$

tasks/main.yml 内で loop ディレクティブを使用するのは問題ありません。

---
- name: Example of using roles
  hosts: all

  tasks:
    - name: The ansible.builtin.import_role module with the loop directive set
      ansible.builtin.import_role:
        name: loop-test2

tasks/main.yml

---
- name: Display the start message
  ansible.builtin.debug:
    msg: "Start!"

- name: Display the contents of the variable item
  ansible.builtin.debug:
    msg: "The value of item is {{ item }}."
  loop:
    - 1
    - 2
    - 3

- name: Display an exit message
  ansible.builtin.debug:
    msg: "End!"

実行ログです。ロール内で loop ディレクティブを設定したタスクが実行されました。
