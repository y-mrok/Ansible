AWX をデプロイ¶

CentOS 8.4 に AWX 19.2.2 をインストールします。

Note

  • AWX をデプロイするにあたり、下記のサイトを参考にしました。ありがとうございました。

    • Install AWX on K3S

    • ansible/awx-operator - GitHub

  • インストール前に（内部）DNS サーバーや hosts ファイルなどで awx.example.jp の名前解決ができるようにしてください。

実行コマンド¶

#
# 既存パッケージを更新
#
dnf -y install open-vm-tools
dnf -y install epel-release
dnf -y update
reboot

#
# OS を設定
#
dnf -y install setroubleshoot-server curl lsof wget
sed -i  '/swap/d' /etc/fstab
swapoff -a
firewall-cmd --permanent --zone=public --add-service=https
firewall-cmd --zone=public --add-masquerade --permanent
firewall-cmd --reload
reboot

#
# K3S をセットアップ
#
curl -sfL https://get.k3s.io | sh
cat /etc/systemd/system/k3s.service
systemctl status k3s.service
kubectl get nodes
kubectl get pods --all-namespaces

#
# AWX operator をデプロイ
#
kubectl config set-context --current --namespace=default
kubectl apply -f https://raw.githubusercontent.com/ansible/awx-operator/0.12.0/deploy/awx-operator.yaml
kubectl get events -w --all-namespaces
kubectl get pods
kubectl logs -f deployment.apps/awx-operator

#
# AWX をデプロイ
#
kubectl create namespace awx
kubectl config set-context --current --namespace=awx
vi myawx.yml
cat myawx.yml
kubectl apply -f myawx.yml
kubectl get events -w --all-namespaces
kubectl logs -f deployment.apps/awx-operator -n default
kubectl get pods
kubectl get svc

#
# admin アカウントのパスワードを生成
#
kubectl get secret awx-admin-password -o jsonpath='{.data.password}' | base64 --decode

myawx.yml ファイルの内容です。

---
apiVersion: awx.ansible.com/v1beta1
kind: AWX
metadata:
  name: awx
spec:
  ingress_type: Ingress
  route_tls_termination_mechanism: edge
  hostname: awx.example.jp

    項目

    入力値

    ユーザー名

    Admin

    パスワード

    SXISjPJsoFP3yre0voCHWjt7ODCIdfil
